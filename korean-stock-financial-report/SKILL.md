# Korean Stock Financial Report

## What this skill does

이미 조사된 한국 상장사 내용을 보기 좋은 보고서형 결과물로 포장한다. 핵심은 "분석을 더 하는 것"보다 이미 확보한 분석을 읽기 좋고 붙여넣기 좋게 다듬는 것이다.

vibe-investing의 `financial-report` 흐름을 가져와 결론을 앞에 두고, 주장과 근거를 분리하고, 숫자/공시/뉴스/체크포인트를 한 흐름으로 묶는다. 내부 검토, 미팅 브리프, 옵시디언 저장, 제안서 초안에 바로 옮겨붙이기 좋은 형식을 목표로 한다.

## When to use

- "방금 조사한 내용을 보고서처럼 정리해줘"
- "이 종목 브리프를 제안서 붙이기 좋게 다듬어줘"
- "총평 먼저 나오고 핵심만 짧게 보이게 써줘"
- "길게 말고 실무형 투자 브리프처럼 정리해줘"

## When not to use

- 아직 팩트 수집이 안 끝난 상태
- 차트 생성이 핵심인 요청
- 실시간 장중 대응 메모
- 순수 valuation 모델링 작업

## Inputs

- 이미 조사된 종목 분석 노트
- 또는 `korean-listed-company-analysis` / `damodaran-korean-stock-bridge` 결과
- 있으면 문서 목적
  - 내부 검토
  - 제안서 첨부
  - 미팅 브리프
  - 옵시디언 저장

## Recommended source order

1. `korean-listed-company-analysis`
   - 기본 사업, 공시, 가격 기대치 정리본 확보
2. `damodaran-korean-stock-bridge`
   - 스토리/기대/조건/파손 조건 보강
3. 필요 시 `k-dart`, `web-search-exa`
   - 누락 팩트나 최근 업데이트만 추가 확인

## Core structure

반드시 아래 순서를 우선한다.

1. 총평
2. 핵심 포인트 3~5개
3. 회사/종목 한 줄 정체
4. 확인된 팩트
5. 해석과 기대치
6. 체크포인트
7. 리스크
8. 한 줄 결론

## Writing rules

- 총평을 맨 위에 둔다.
- 표보다 짧은 불릿을 우선한다.
- 숫자는 너무 많이 늘어놓지 말고 주장에 필요한 것만 남긴다.
- 공시 날짜는 가능하면 유지한다.
- 기사/루머/해석은 팩트와 분리해서 쓴다.
- 딱딱한 증권사 문체보다 바로 읽히는 실무형 한국어를 우선한다.

## Variants

### A. 아주 짧은 브리프

아래 5블록만 남긴다.
- 총평
- 핵심 포인트
- 확인된 팩트
- 체크포인트
- 한 줄 결론

### B. 제안서/회의용

- 총평 2줄
- 회사 정체 2~3줄
- 왜 지금 보는지 2줄
- 리스크 1~2줄
- 마지막에 미팅용 질문 2개 추가 가능

### C. 옵시디언 저장용

- 제목에 회사명 + 날짜
- 소스 링크나 공시명은 하단 참고에 따로 둔다.
- 본문은 읽기 중심으로 짧게 유지한다.

## Failure modes

- 분석 단계와 보고서 포장 단계를 섞으면 품질이 떨어진다. 팩트가 비어 있으면 먼저 분석 스킬로 돌아간다.
- 숫자를 너무 많이 넣으면 읽기성이 망가진다. 주장에 필요한 숫자만 남긴다.
- 기사와 공시를 같은 무게로 섞지 않는다.
- 너무 딱딱하고 길게 쓰면 활용도가 떨어진다.

## Done when

- 총평이 맨 위에 있다.
- 핵심 포인트가 3~5개 이내다.
- 회사 정체가 한두 줄로 정리돼 있다.
- 팩트와 해석이 섞이지 않았다.
- 체크포인트와 리스크가 따로 보인다.
- 마지막 한 줄 결론이 남아 있다.

## Notes

- 분석용 스킬이 아니라 포장용 스킬이다.
- `gomgom_obsidian`, `powerpoint` 같은 후속 문서화 흐름과 잘 맞는다.
- 투자 조언처럼 단정하지 않고 브리프 톤을 유지한다.
