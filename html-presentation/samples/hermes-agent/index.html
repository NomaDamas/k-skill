    왜 Hermes인가

    7가지 핵심 차별점

    기존 CLI 에이전트가 갖지 못한 것

    • Real terminal interface — 멀티라인 편집·슬래시 자동완성·스트리밍 출력이 갖춰진 완전한 TUI
    • Lives where you do — Telegram·Discord·Slack·WhatsApp·Signal·CLI, 단일 gateway 프로세스
    • Closed learning loop — 사용 중 스킬 자가 개선, 세션 간 기억 영속화, 사용자 모델 심화
    • Scheduled automations — 내장 cron으로 자연어 일정 자동화(리포트·백업·감사)
    • Delegates & parallelizes — 격리된 서브에이전트로 병렬 작업, zero-context-cost 파이프라인
    • Runs anywhere — $5 VPS부터 GPU 클러스터까지, 7개 terminal backend 지원

    기존 에이전트 vs Hermes

    기존

    • 노트북에 묶임
    • 세션 종료 시 기억 소실
    • 도구는 정적, 수동 업데이트
    • CLI 전용
    • 매번 같은 실수를 반복

    Hermes

    • 어디서나 실행 ($5 VPS ~ GPU 클러스터)
    • 세션 간 사용자 모델 심화
    • 사용 중 스킬 자가 개선
    • 6개 메시징 플랫폼 + CLI
    • 자신의 과거 대화를 검색하고 회상

    닫힌 학습 루프

    Closed Learning Loop

    • 1. 경험 → 스킬: 복잡한 작업 후 에이전트가 자율적으로 스킬을 생성한다
    • 2. 사용 → 개선: 스킬은 호출될 때마다 스스로 다듬어진다
    • 3. 영속화 nudge: 에이전트가 직접 메모리 큐레이션을 자극한다
    • 4. 회상: FTS5 세션 검색 + LLM 요약으로 과거를 재해석한다

    60초 설치 (Linux · macOS · WSL2)

    # 1. 설치 (curl 한 줄)
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash

# 2. 쉘 리로드
source ~/.bashrc

# 3. 시작
hermes

    어디서나 실행

    7개 terminal backend, 1개 명령

    • local — 로컬 머신
    • Docker — 격리 컨테이너
    • SSH — 원격 서버
    • Modal / Daytona — 서버리스 (유휴 시 거의 0원)
    • Singularity — HPC/GPU 클러스터
    • Vercel Sandbox — 에지 실행

    CLI vs 메시징 플랫폼

    터미널(CLI)

    • hermes 즉시 시작
    • 멀티라인 편집
    • 슬래시 커맨드 자동완성
    • Ctrl+C로 인터럽트 및 방향 전환
    • 스트리밍 도구 출력

    메시징 게이트웨이

    • Telegram · Discord · Slack
    • WhatsApp · Signal
    • 음성 메모 자동 전사
    • 크로스 플랫폼 대화 연속성
